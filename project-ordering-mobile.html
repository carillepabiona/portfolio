Project Overview

Problem · Solution · Outcome

Problem

Restaurant staff relied on manual paper-based order taking, causing errors, slow kitchen communication, and no visibility into real-time order status — leading to poor customer experience and operational inefficiencies.

Solution

Built a cross-platform mobile application using .NET MAUI Hybrid Blazor that digitizes the entire ordering workflow — from customer menu browsing and order placement to POS processing and live Kitchen Display System updates.

Outcome

Eliminated manual order errors, improved kitchen-to-floor communication, and provided a smooth end-to-end digital restaurant experience. Delivered as a fully functional capstone project demonstrating real-world mobile development skills.

How It Was Built

Development Timeline

Planning & Requirements

Gathered requirements, defined project scope, identified core modules, and outlined system architecture.

System & UI Design

Designed wireframes and UI layouts for all screens. Established the design system and navigation flow.

Database Design

Designed the relational schema using MariaDB for server-side data and SQLite for local offline storage.

Development

Built POS, KDS, and menu ordering modules using .NET MAUI Hybrid Blazor with C# backend logic and Blazor components.

Testing & Debugging

Conducted functional testing across all modules, fixed edge-case bugs, and validated real-time order sync behavior.

Deployment & Presentation

Packaged and deployed the app for capstone presentation. Demonstrated all workflows to the panel.

Learnings

Challenges & Solutions

Challenge

Real-time order synchronization between POS and KDS.

Solution

Implemented database-driven polling with MariaDB to push order status updates to the KDS in near real-time.

Challenge

Bridging .NET MAUI native layer with Blazor web components.

Solution

Used .NET MAUI Hybrid Blazor's WebView integration to seamlessly share Razor components between native and web layers.
