Project Overview

Problem · Solution · Outcome

Problem

Organizations relied on physical filing cabinets and paper-based records, making document retrieval slow, prone to loss, and lacking in access control — creating security risks and inefficiencies in record management.

Solution

Developed a web-based archiving system using ASP.NET Core Razor Pages that digitizes, stores, and manages organizational documents with role-based access control, audit logging, and fast keyword-based retrieval.

Outcome

Eliminated manual filing processes, reduced document retrieval time significantly, and enforced access security through user roles. Delivered a fully functional academic project with a clean, professional interface.

How It Was Built

Development Timeline

Planning & Requirements

Identified system goals, defined module scope, and planned database schema based on document management needs.

System & UI Design

Designed page layouts using Razor Pages, created navigation flows, and established UI components with Bootstrap.

Database Design

Designed SQL Server schema with tables for documents, users, roles, categories, and audit logs. Configured EF Core migrations.

Development

Built all modules — document archiving, access control, user management, and audit logging — with ASP.NET Core and EF Core.

Testing & Security Review

Tested all CRUD operations, validated access restrictions per role, and verified audit log accuracy across all actions.

Learnings

Challenges & Solutions

Challenge

Implementing granular role-based access across all document actions.

Solution

Leveraged ASP.NET Core's built-in authorization with custom policies to restrict actions per role at both the controller and view levels.

Challenge

Efficiently storing and retrieving large volumes of document metadata.

Solution

Designed indexed SQL Server tables and used EF Core LINQ queries optimized for filtered, paginated result sets.
